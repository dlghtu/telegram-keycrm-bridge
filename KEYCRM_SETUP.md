# Усе в KeyCRM: чати + замовлення + ТТН однією кнопкою

## Архітектура (як має працювати для клієнта)

```text
Клієнт пише в Telegram-бота
        ↓
KeyCRM «Чати»  ← менеджер відповідає ТУТ
        ↓
З чату / картки: створити замовлення в CRM
        ↓
Кнопка «Створити ТТН» (Нова Пошта всередині KeyCRM)
        ↓
Клієнт отримує ТТН (чат CRM / SMS / опційний webhook-сервіс)
```

**Не потрібен** Make/Zapier і **не потрібен** окремий Telegram-webhook на Render для чатів.

Зараз у бота webhook уже на KeyCRM:
`https://messaging.keycrm.app/webhooks/telegram/...` — це правильно для живих чатів.

---

## Крок 1. Telegram-бот у KeyCRM (чати) — часто вже зроблено

1. **Налаштування → Комунікації → Чати / SMS / Email**
2. **Додати канал** (або перевір, що канал уже є)
3. Тип: **Telegram Bot**
4. **API ключ** = токен від @BotFather
5. Зберегти

Документація:  
https://help.keycrm.app/ru/communications-sms-telegram-viber-instagram-email/kak-podkliuchit-telegram-bota-v-key-crm-dlia-administratora

**Тест:** клієнт пише боту → повідомлення з’являється в **Чати** KeyCRM → менеджер відповідає з CRM.

⚠️ Бот має бути підключений **тільки** до KeyCRM.  
Не викликай `setWebhook` на `telegram-keycrm-bridge.onrender.com` — чати в CRM зламаються.

---

## Крок 2. Нова Пошта → ТТН «в один клік»

1. **Налаштування → Доставки**
2. **Додати службу доставки** → **Нова Пошта**
3. Ввести **API-ключ** з кабінету Нової Пошти  
   (НП: Налаштування → Безпека → API ключі)
4. Обрати відправника / склад за замовчуванням, зберегти

Документація:  
https://help.keycrm.app/ru/carrier-integrations/kak-nastroit-ghienieratsiiu-ttn-v-crm-instruktsiia-dlia-administratora

### Як зробити ТТН у замовленні

1. Відкрий замовлення (з чату: «оформити замовлення» / нова картка)
2. Заповни: ПІБ, телефон, місто, відділення НП, товари
3. Біля поля **Трекінг-код** натисни кнопку створення ТТН  
   (або кнопку створення накладної в списку замовлень)
4. Підтверди параметри → ТТН з’явиться в замовленні і в кабінеті НП

Документація:  
https://help.keycrm.app/ru/carrier-integrations/kak-dobavit-ttn-k-sushchiestvuiushchiemu-zakazu

Відео: «Як підключити Нову Пошту та робити ТТН в один клік в KeyCRM».

---

## Крок 3. Замовлення з чату

У відкритому чаті з клієнтом у KeyCRM:

- створити **картку покупця** (якщо ще немає);
- **створити замовлення** прямо з діалогу / прив’язати існуюче;
- додати товари зі складу (SKU) — тоді CRM спише/зарезервує залишки;
- доставка: Нова Пошта + місто + відділення.

Після цього — крок 2 (ТТН).

---

## Крок 4. Повідомити клієнта про ТТН

### Варіант A (найкращий) — усередині KeyCRM

1. **Налаштування → джерела / канали продажів** (джерело «Telegram bot»)
2. Увімкни **відправку трек-номера** клієнту (`send_tracking_code`), якщо є в UI
3. Або **Автоматизації**: коли з’явився tracking_code → надіслати повідомлення в чат Telegram

### Варіант B (наш Render) — запасний webhook

URL вебхука в KeyCRM (Автоматизації / Webhooks):

```text
https://telegram-keycrm-bridge.onrender.com/webhooks/keycrm?secret=kcrm_wh_sec_change_me_7b1c
```

Подія: зміна замовлення / додано трекінг.

Щоб сервіс знав, **кому** писати в Telegram, один із варіантів:

- у `manager_comment` замовлення рядок: `tg_chat_id:123456789`
- або custom field з chat_id
- або POST `https://telegram-keycrm-bridge.onrender.com/map?secret=...`  
  body: `{"orderId":152,"chatId":"123","phone":"+380..."}`

Якщо в CRM вже йде повідомлення з ТТН у чат — **варіант B не обов’язковий**.

## Робочий день менеджера (продай це клієнту)

| Дія | Де |
|-----|-----|
| Відповісти клієнту | KeyCRM → **Чати** |
| Оформити замовлення | KeyCRM → з чату / **Замовлення** |
| Створити ТТН | Замовлення → кнопка **ТТН / трекінг** |
| Надіслати ТТН клієнту | Автоматика CRM або чат вручну |

Усе в **одній системі**. Render — лише опційна «обгортка», не головний UI.

---

## Render (companion) — статус

- Health: https://telegram-keycrm-bridge.onrender.com  
- Не використовуй його як Telegram webhook для вхідних повідомлень.
- Environment: `TELEGRAM_BOT_TOKEN`, `KEYCRM_API_TOKEN`, `KEYCRM_WEBHOOK_SECRET` (для опційного TTN-notify).
